## 修复照片墙刷新卡顿（第三次）
- **原因分析**：
  1. `loading="lazy"` + 换 src 触发懒加载延迟
  2. `onload` 在解码完成前就 resolve，大 PNG 解码时仍闪
  3. 132 张图同时换 src 触发级联重排，干扰 CSS transform 动画
- **修复**：
  1. `onload` → `Image.decode()`（仅在完全解码后 resolve）
  2. 去掉 `loading="lazy"`
  3. `contain: layout style paint` 隔离每个 item 的重排
  4. `<img width="300" height="200">` 固定尺寸防 layout shift
  5. 所有 swap 收敛到单个 `requestAnimationFrame` 帧
  6. track 保持 `will-change: transform` 跑 GPU 合成层

## 修复照片墙瞬间闪黑（第四次 — 双缓冲方案）
- **根因**：`img.src = newSrc` 赋值时浏览器必然经历「清空旧图 → 渲染新图」周期，清空瞬间显示黑底，即使图片已预解码也无法跳过
- **根治**：双缓冲（double-buffer）架构
  1. 每行渲染两套完整 track（buffer A / buffer B），绝对定位重叠
  2. 可见 buffer 有 `opacity: 1` + `pointer-events: auto`，隐藏 buffer 有 `opacity: 0` + `pointer-events: none`
  3. 换图只在隐藏 buffer 上操作（用户看不到任何变化）
  4. 全部换完后 CSS `transition: opacity 0.6s` 交叉淡入淡出
  5. 两套 track 的 CSS 动画永远独立运行，互不干扰
  6. hover 只作用于可见 buffer
